# MVP Patterns for One-Person Companies

## Pattern 1: Free → Content → Paid (服务型OPC冷启动)

最适合：咨询服务、定制服务、AI赋能等"卖能力"型OPC

**三步走：**

### 第一步：免费诊断（第1周）
- 找3个目标客户做免费服务
- 目标：验证交付流程可行 + 收集反馈和证言
- 产出：诊断报告 + 客户证言 + 流程SOP初稿

### 第二步：内容发酵（第2周）
- 把诊断过程转化为内容（脱敏案例、干货、踩坑经验）
- 目标：验证内容能否吸引目标客户关注
- 产出：第一批粉丝 + 咨询线索

### 第三步：付费转化（第3周）
- 从咨询线索中转化第1个付费客户
- 目标：验证付费意愿 + 完成商业闭环
- 产出：第1笔收入 + 完整案例

**优势：**
- 一石二鸟：找客户和建IP同时进行
- 风险最低：先验证再投入
- 自然裂变：免费客户成为证言，内容带来新线索

**适用场景：**
- 缺乏案例和信任的新服务
- 需要建立个人IP的领域
- 目标客户分散，需要内容获客

## Pattern 2: 内容先行（知识型OPC）

适合：课程、社群、知识付费

**流程：**
1. 先发30天内容，测试市场反应
2. 根据反馈确定产品方向
3. 预售验证付费意愿
4. 正式交付

---

## Pattern 3: 工具原型（产品型OPC）

适合：SaaS、工具、自动化

**流程：**
1. 用NoCode/低代码搭建MVP
2. 找10个种子用户免费试用
3. 根据反馈迭代
4. 定价上线

---

## 选择建议

| 你的类型 | 推荐Pattern |
|---------|-------------|
| 卖服务/咨询 | Pattern 1 (Free→Content→Paid) |
| 卖知识/课程 | Pattern 2 (内容先行) |
| 卖工具/产品 | Pattern 3 (工具原型) |
